Tutorial

Build your own DIMS dashboard

What your data needs to look like :
  • Group files by session. Everything from one recording shares a single session ID — the video, its time series, and its transcript.
  • Video — an .mp4.
  • Time series — a .csv with a Time column (in seconds, case-sensitive) plus at least one measurement column. One CSV per data type.
  • Transcript (optional) — JSON shaped as { "segments": [ {start, end, speaker, text} ] }.
  • ELAN (optional) — an .eaf file.
Most important: the Time axis is mapped directly onto the video clock, so a session's video length and time-series length should match. If they differ, the video and signals won't line up (dead space, or the signal cut off).

For non-coders — the Dashboard Builder

A point-and-click app: drop your files in, click through a wizard, get a ready-to-deploy dashboard. No JSON editing, no file renaming, no terminal scripts.

One prerequisite: you need Python 3.9+ installed (a free, one-time install from python.org/downloads). That's the only thing — no git, no internet required once you have the builder.

  1. Download the builder

    ⬇ Download the builder (ZIP)

    Unzip it — no git needed. The DIMS dashboard code is bundled inside, so there's nothing else to download.

  2. Start it — one click

    Double-click the launcher in the unzipped folder. It installs everything the first time and then opens the wizard in your browser:

    • macOS: run.command
    • Windows: run.bat
    • Linux: run.sh
    macOS — first time only: you may see “Apple could not verify ‘run.command’ is free of malware.” Click Done (not Move to Trash), then open System Settings → Privacy & Security, scroll to Security, and click Open Anyway next to run.command. This only happens once. (It's macOS flagging any downloaded, unsigned script — not a problem with the builder.)

    Prefer the terminal? That also avoids the macOS prompt above — open a terminal in the unzipped folder and run:

    pip install -r requirements.txt
python builder.py

  3. Create a project

    Choose an empty output folder and a title/authors. The builder uses its built-in copy of the template — no internet or git required. (Advanced users can opt to clone the latest from GitHub instead.)

  4. Add your files

    Drag in your .mp4 video, your time-series .csv files, your _transcript.json, and optionally an ELAN .eaf. The builder recognises each file, groups them under a session, and checks them as you go — so problems surface before you build.

  5. Choose optional analyses

    Toggle RQA, cross-wavelet (needs at least two data types), and ELAN annotations. The wizard explains what each tab shows.

  6. Build & precompute

    Build writes config.json and copies your files into the exact assets/ layout. Precompute then runs the RQA / cross-wavelet scripts for you in a project-local Python environment — progress streams live.

  7. Preview & deploy

    Preview opens the finished dashboard locally. The output folder is self-contained and ready to host — the builder shows copy-paste commands for GitHub Pages, Netlify, or Vercel.

For coders — clone the template & edit by hand

Full control: start from the template, drop files into assets/ with the right names, edit config.json, and (optionally) run the analysis scripts.

You'll need: git and Python 3.9+ (python.org/downloads) — the latter for the optional precompute and the local preview server.

  1. Create your repo from the template

    On DIMS_dashboard_template, click “Use this template” → create a new repo — or just clone it.

    git clone https://github.com/dims-network/DIMS_dashboard_template my-dashboard
cd my-dashboard

  2. Add your assets with the exact names

    File names are a strict convention — the dashboard finds your data by name. For a session id (e.g. session1) and a data type (e.g. bodysync):

    assets/videos/session1.mp4 assets/timeseries/session1_bodysync.csv # Time + one measurement column assets/timeseries/session1_neuralsync.csv assets/transcripts/session1_transcript.json # { "segments": [ {start,end,speaker,text} ] } assets/elan/session1.eaf # optional (ELAN)
    Heads up: the names must match exactly — {videoID}_{dataType}.csv, {videoID}.mp4, {videoID}_transcript.json. A missing Time column or a mismatched name is the most common reason a tab stays blank.

  3. Edit config.json

    Declare your sessions, their data types, which analyses to enable, and the page text:

    {
  "videoIDs": ["session1"],
  "dataTypes": { "session1": ["bodysync", "neuralsync"] },
  "include_RQA": ["bodysync", "neuralsync"],
  "include_crosswavelet": ["bodysync", "neuralsync"],
  "include_elan": false,
  "defaultWindowSize": 5,
  "title": "My DIMS Dashboard",
  "subtitle": "Dynamic Interaction and Multimodal Signals",
  "authors": "Your name(s)",
  "contacts": "you@example.org"
}

    Cross-wavelet compares pairs, so it needs at least two data types. Every name in include_RQA / include_crosswavelet must have a matching CSV.

  4. (Optional) Precompute the analyses

    RQA and cross-wavelet read your CSVs and write JSON the dashboard loads. Run them once whenever those toggles are on:

    pip install -r opt/requirements.txt
python opt/step_RQA.py          --config config.json --output-dir assets/rqa
python opt/step_crosswavelet.py --config config.json --output-dir assets/crosswavelet

    This produces assets/rqa/{videoID}_rqa_data.json and assets/crosswavelet/{videoID}_crosswavelet_data.json.

  5. Preview locally

    Use the bundled server (it supports HTTP Range requests, which the video needs for seeking):

    python serve.py 8000   # then open http://localhost:8000

  6. Deploy

    Push to GitHub and enable Pages (branch main, folder /root), or host on any static host. Just make sure it serves Range requests so video seeking works.
