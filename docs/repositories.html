Repositories

What each repository is, how it is laid out, and how it relates to the canonical template. See Architecture for the overall model.

DIMS_dashboard_template

The canonical dashboard code — the single source of truth. All code changes happen here.

js/ # dashboard frontend (app.js, video-component.js) css/ # styles opt/ # analysis scripts: step_RQA.py, step_cRQA.py, step_crosswavelet.py index.html # dashboard entry point serve.py # local preview server (HTTP Range support for video seeking) config.json # empty/default config (no data) assets/ # empty skeleton: videos/ timeseries/ transcripts/ elan/ .github/ # Pages deploy workflow (static.yml)

Repo: github.com/dims-network/DIMS_dashboard_template

DIMS_Dashboard

The canonical, deployed dashboard — the template code plus real research data, published to GitHub Pages. It doubles as the integration test bed: new template features are tried here against real data before being blessed.

  • Dashboard code lives at the repo root (mixed with data), so it consumes the template as an upstream remote: git merge template/main.
  • Its own data is config.json + assets/; a .gitattributes rule keeps config.json on merge.
  • Update it with scripts/update-from-template.sh.

Repo: github.com/dims-network/DIMS_Dashboard · Live: dims-network.github.io/DIMS_Dashboard

DIMS_dashboard_builder

The no-code wizard. A small Flask app that scaffolds a dashboard project, places assets under the strict layout, writes config.json, and runs the template's analysis scripts — so a non-technical user needs neither git nor a network connection.

app/ # Flask app: server.py, project.py, media.py, precompute.py, validate.py app/static/ # wizard UI (index.html, builder.js) template/ # vendored copy of DIMS_dashboard_template (a git subtree) scripts/ # update-template.sh (subtree pull) builder.py # launcher
  • The template/ directory is a git subtree of the template repo — the files are committed (offline-safe) but their history tracks upstream. Update with scripts/update-template.sh.
  • app/project.py acquire_template() copies the template into each generated project and refreshes its code on rebuild while preserving the user's data.

Repo: github.com/dims-network/DIMS_dashboard_builder

Python / Dash viewers & forks

Outside the static-dashboard family:

  • DIMS_Dashboard_Ortho — a study-specific fork of the dashboard (multi-perspective video + trajectory tab).
  • bots-viewer — a Dash app for a two-player grid game (trajectory playback + RQA/cRQA).
  • Ortho-viewer — a Dash viewer embedding a DIMS sub-dashboard for ORTHO game sessions.
