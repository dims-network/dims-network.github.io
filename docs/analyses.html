Analyses

Three optional analyses are precomputed from your time series into JSON that the dashboard loads. Each is a standalone script under opt/, reads config.json, and writes one file per video (see Data & config model).

Running them

The builder runs these for you. By hand (from a project root):

pip install -r opt/requirements.txt
python opt/step_RQA.py          --config config.json --output-dir assets/rqa
python opt/step_crosswavelet.py --config config.json --output-dir assets/crosswavelet
python opt/step_cRQA.py         --config config.json --output-dir assets/crqa

RQA — Recurrence Quantification Analysis

Recurrence within a single signal: how often the signal revisits similar states over time. Each series is z-normalized; a distance matrix is thresholded to a target recurrence rate (~7%) to produce a recurrence plot, downsampled to ≤500×500 for the browser. The dashboard renders the recurrence plot with the signal as marginal traces, plus windowed metrics.

Windowed metrics (a sliding window along the diagonal): recurrence rate (RR), determinism (DET), laminarity (LAM), and longest diagonal line (L_MAX).

Cross-RQA

Recurrence between two signals — when states of signal A recur in signal B. The two series are interpolated onto a common time grid and z-normalized; the cross-recurrence matrix is thresholded the same way. The dashboard renders the cross-recurrence plot with signal 1 as the top marginal and signal 2 as the left marginal (matching the RQA layout), plus the same windowed RR/DET/LAM/L_MAX metrics.

Output shape: crqa_data[pair].visualization carries time, data_x, data_y, matrix_size and a sparse sparse_matrix; windowed_metrics carries the RR/DET/LAM/L_MAX series.

Cross-wavelet

Time–frequency coupling between two signals, via the continuous wavelet transform (Morlet) using pycwt. Produces cross-wavelet power, coherence, and phase over time and period, with AR1-based significance and a cone of influence (COI). Results are downsampled for the browser.

Output shape: crosswavelet_pairs[pair].visualization carries time, period/freqs, power, coherence, phase, coi, and significance fields.
Pairs vs. flat list: RQA takes a flat list of data types; cross-wavelet and cross-RQA take pairs. In the builder you pick exactly which pairs to compute; by hand, list them as [["a","b"], ...] in config.json.
