Data & config model

A dashboard is driven by one config.json plus a strict assets/ file-naming convention. The builder writes both for you; this page is the underlying contract.

Asset layout

Files are found by name. Everything from one recording shares a single session ID ({videoID}); a time series also has a data type ({dataType}).

assets/videos/{videoID}.mp4 assets/timeseries/{videoID}_{dataType}.csv # Time column (seconds) + one measurement column assets/transcripts/{videoID}_transcript.json # { "segments": [ {start,end,speaker,text} ] } assets/elan/{videoID}.eaf # optional (ELAN)

Analysis outputs are written alongside, one file per video:

assets/rqa/{videoID}_rqa_data.json assets/crqa/{videoID}_crqa_data.json assets/crosswavelet/{videoID}_crosswavelet_data.json
Most common cause of a blank tab: a mismatched file name or a missing time column. The time column is matched case-insensitively (time, Time, TIME all work) and is in seconds; it is mapped onto the video clock, so a session's video length and time-series length should match.

config.json schema

The dashboard reads exactly these keys:

{
  "videoIDs": ["session1"],                                # sessions in the dashboard
  "dataTypes": { "session1": ["bodysync", "neuralsync"] }, # data types per session
  "include_RQA": ["bodysync", "neuralsync"],              # flat list of data types
  "include_crosswavelet": [["bodysync", "neuralsync"]],   # list of PAIRS
  "include_cRQA": [["bodysync", "neuralsync"]],           # list of PAIRS
  "include_elan": false,
  "defaultWindowSize": 5,                                 # seconds, time-window slider default
  "title": "My DIMS Dashboard",
  "subtitle": "Dynamic Interaction and Multimodal Signals",
  "authors": "Your name(s)",
  "contacts": "you@example.org"
}
KeyTypeNotes
videoIDsstring[]session IDs; each needs assets named after it.
dataTypesobjectvideoID → list of its data types.
include_RQAstring[]data types to run recurrence analysis on.
include_crosswaveletpair[]each pair ["a","b"]; a flat list is also accepted and expanded to all pairs.
include_cRQApair[]each pair ["a","b"].
include_elanboolshow the ELAN annotation tab.
defaultWindowSizenumberdefault analysis window, in seconds.
title / subtitlestringpage heading.
authors / contactsstringcredit / contact line.
Source of truth: the contract is defined by the dashboard (the template) — config.json is read by the frontend (js/app.js)
